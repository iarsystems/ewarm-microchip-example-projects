00030 /**
00031  * \file
00032  *
00033  * SkipBlockNandFlash layer supplies application a set of interface to operate nandflash.
00034  * which include initialize, block erase, block write/read, page write/read. \n
00035  * This layer is to just skip over the bad blocks and place the data in the known good blocks.
00036  * The algorithm starts by reading the entire spare area of the entire memory. The addresses
00037  * of the factory-marked bad blocks are then collected in the programmer RAM. Next, the image is
00038  * sequentially programmed (page by page) into the target device. When the target address
00039  * corresponds to a bad block address, these pages are stored in the next good block, skipping
00040  * the bad block.
00041  * The SkipBlocks method is a very generic and well-performing strategy. When a bad block is
00042  * encountered, the algorithm simply skips ahead to the next good block.
00043  */
