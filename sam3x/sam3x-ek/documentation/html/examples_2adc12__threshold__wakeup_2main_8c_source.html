00030 /**
00031  *  \page adc12_threshold_wakeup ADC12 Threshold Wakeup Example
00032  *
00033  *  \section Purpose
00034  *
00035  *  The adc12_threshold_wakeup example demonstrates how to use ADC with
00036  *  threshold wakeup.
00037  *
00038  *  \section Requirements
00039  *
00040  *  This package can be used with SAM3X-EK. To enable full scale measurement
00041  *  of the potentiometer by default configuration. Close jumper JP4 on 1 and 2,
00042  *  the outer pins of it.
00043  *
00044  *  \section Description
00045  *  This example uses TIOA0 as external trigger instead of software trigger for
00046  *  ADC conversion. The TIOA0 is a 1ms period square wave. The rising edge
00047  *  during each period would trigger the ADC to start a conversion on given channel
00048  *  which is connected to the potentiometer. This example shows a menu as below
00049  *  upon running,
00050  *  \code
00051  *  -- Menu Choices for this example--
00052  *  -- 0: Display voltage on potentiometer.--
00053  *  -- 1: Display thresholds.--
00054  *  -- 2: Modify low threshold.--
00055  *  -- 3: Modify high threshold.--
00056  *  -- 4: Choose comparison mode.--
00057  *  -- i: Display ADC information.--
00058  *  -- m: Display this main menu.--
00059  *  -- s: Enter sleep mode.--
00060  *  \endcode
00061  *  With the user interface, comparison window and mode could be set. The ADC
00062  *  supports 4 kinds of comparison events as following:
00063  *
00064  *  - lower than the lower threshold.
00065  *  - higher than the high threshold.
00066  *  - in the comparison window.
00067  *  - out of the comparison window.
00068  *
00069  *  If the target got an 'S' or 's' from user's input,the core fell in sleep
00070  *  by __WFI.Tuning the potentiometer to enable the ADC input fall into the
00071  *  comparison window and then generate a trigger event. The comparison event
00072  *  will wake the system up.
00073  *
00074  *  \section Usage
00075  *
00076  *  -# Build the program and download it inside the evaluation board. Please
00077  *     refer to the
00078  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00079  *     SAM-BA User Guide</a>, the
00080  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00081  *     GNU-Based Software Development</a>
00082  *     application note or to the
00083  *     <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00084  *     IAR EWARM User Guide</a>,
00085  *     depending on your chosen solution.
00086  *  -# On the computer, open and configure a terminal application
00087  *     (e.g. HyperTerminal on Microsoft Windows) with these settings:
00088  *    - 115200 bauds
00089  *    - 8 bits of data
00090  *    - No parity
00091  *    - 1 stop bit
00092  *    - No flow control
00093  *  -# In the terminal window, the
00094  *     following text should appear (values depend on the board and chip used):
00095  *     \code
00096  *      -- ADC12 Threshold Wakeup Example xxx --
00097  *      -- xxxxxx-xx
00098  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00099  *      -- Menu Choices for this example--
00100  *      -- 0: Display voltage on potentiometer.--

00104  *      -- 4: Choose comparison mode.--
00105  *      -- i: Display ADC information.--
00106  *      -- m: Display this main menu.--
00107  *      -- s: Enter sleep mode.--
00108  *     \endcode
00109  *  -# Input the command according to the menu.
