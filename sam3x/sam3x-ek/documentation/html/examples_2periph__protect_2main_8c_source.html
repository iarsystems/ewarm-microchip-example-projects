00031 /**
00032  *  \page periph_protect Peripheral Protect Example
00033  *
00034  *  \section Purpose
00035  *
00036  *  The peripheral_protect example demonstrates how to prevent a program from
00037  *  corrupting a PIO controller behavior.
00038  *
00039  *  \section Requirements
00040  *
00041  *  This package can only be used with SAM3X-EK.
00042  *
00043  *  \section Description
00044  *
00045  *  The application shows the protective mechanism of the PIO controller.
00046  *  The example chooses a pin connected with a led (D2, PA12). This pin is
00047  *  multiplexed with the PWM0 of the peripheral PWM. In the example, the control
00048  *  of the pin could be switched by pressing the pushbutton 1. When the pin is
00049  *  controlled by PIOC, the connected LED is turned on. When the pin is used as
00050  *  PWM0, the connected LED is keeping flashing.
00051  *  The switching is implemented by writing ABCD Select Registers. This
00052  *  register can only be written if the WPEN bit is cleared in PIO Write
00053  *  Protect Mode Register. The WPEN bit can be set/clear by pressing the
00054  *  pushbutton 2. If the write protection is enable, switching the peripheral
00055  *  funtions will be unsuccessful. A warning is generated as well. The source
00056  *  of violation will be reported to the console by reading the register,
00057  *  pio write protect status register.
00058  *
00059  *  \section Usage
00060  *
00061  *  -# Build the program and download it inside the evaluation board. Please
00062  *     refer to the
00063  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00064  *     SAM-BA User Guide</a>, the
00065  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00066  *     GNU-Based Software Development</a>
00067  *     application note or to the
00068  *     <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00069  *     IAR EWARM User Guide</a>,
00070  *     depending on your chosen solution.
00071  *  -# On the computer, open and configure a terminal application
00072  *     (e.g. HyperTerminal on Microsoft Windows) with these settings:
00073  *    - 115200 bauds
00074  *    - 8 bits of data
00075  *    - No parity
00076  *    - 1 stop bit
00077  *    - No flow control
00078  *  -# Start application.
00079  *  -# In the terminal window, the
00080  *     following text should appear (values depend on the board and chip used):
00081  *     \code
00082  *      -- PIO Alternate Function Example xxx --
00083  *      -- xxxxxx-xx
00084  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00085  *     \endcode
00086  *  -# Perform erase operation following the application prompt.
00087  *
00088  *  \section References
