00031  *  \page usart_synchronous  USART Synchronous Mode Example
00032  *
00033  *  \section Purpose
00034  *
00035  *  This example demonstrates the Synchronous  mode provided by the USART
00036  *  peripherals on SAM3X.
00037  *
00038  *  \section Requirements
00039  *
00040  *  This example can be used on sam3x evaluation kit and requires 3 fly wires 
00041  *  connected the following pins.
00042  *  - <b> Usart1   --   Usart3</b>
00043  *   - TXD(PA13)   --  RXD(PD5)
00044  *   -  RXD(PA12)  --  TXD(PD4)
00045  *   -  SCK(PA16)  --  SCK(PE16)
00046  *
00047  *  \section Description
00048  *
00049  *  This application gives an example of how to use USART in synchronous mode.
00050  *  Synchronous operations provide a high speed transfer capability. The
00051  *  transfer under this mode needs a pair of master and slave, which is
00052  *  determined by which one offers the clock source.
00053  *
00054  *  The example initialized USART1 as master by default. To enable the
00055  *  communication between each side of the connection,the user should change
00056  *  the mode of another side to slave through user interface. If well configured,
00057  *  transfer could be started by typing 'r' and  'w' from terminal application.
00058  *  This example also leaves the interface to select the clock frequency.
00059  *
00060  *  The meaning of each input character could be found in items of the main menu.
00061  *
00062  *
00063  * \section Usage
00064  *
00065  *  -# Build the program and download it inside the evaluation board. Please
00066  *     refer to the <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">SAM-BA User Guide</a>,
00067  *     the <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">GNU-Based Software Development</a>
00068  *     application note or to the <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">IAR EWARM User Guide</a>,
00069  *     depending on your chosen solution.
00070  *  -# Connect a serial cable to the UART port on the evaluation kit.
00071  *  -# On the computer, open and configure a terminal application (e.g.
00072  *     HyperTerminal on Microsoft Windows) with these settings:
00073  *        - 115200 bauds
00074  *        - 8 data bits
00075  *        - No parity
00076  *        - 1 stop bit
00077  *        - No flow control
00078  *  -# Start the application. The following traces shall appear on the terminal:
00079  *     \code
00080  *     -- USART Synchronous Mode Example  xxx --
00081  *     -- xxxxxx-xx
00082  *     -- Compiled: xxx xx xxxx xx:xx:xx --
00083  *     -- Menu Choices for this example --
00084  *     -- [0-3]:Select clock frequency of master --
00085  *     -- i: Display configuration info
00086  *     -- w: Write data block .--
00087  *     -- r: Read data block.--
00088  *     -- s: Switch between master and slave mode.--
00089  *     -- m: Display this menu again.--
00090  *     --USART1 in MASTER mode--
00091  *
00092  *     \endcode
00093  *  -# The main menu will guide the user to configure the device and conduct
00094  *     operations.

00201  * and if transmission done, set sentDone to true.
00202  *
00203  *----------------------------------------------------------------------------*/
00204 void USART1_IrqHandler(void)
00205 {
00206     uint32_t dwStatus;
00207 
00208     /*  Read USART status */
00209     dwStatus = BOARD_USART1_BASE->US_CSR;
00210     /* Receive buffer is full */
00211     if ((dwStatus & US_CSR_ENDRX) == US_CSR_ENDRX && ucState == USART1_STATE_READ) {
00212         recvDone = true;
00213         USART_DisableIt(USART1, US_CSR_ENDRX);
00214     }
00215     if ((dwStatus & US_CSR_TXBUFE) == US_CSR_TXBUFE && ucState == USART1_STATE_WRITE){
00216         sentDone = true;
