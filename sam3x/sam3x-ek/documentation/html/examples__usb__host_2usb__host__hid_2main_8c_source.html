00024  * \section install Description
00025  * This embedded application source code illustrates how to implement a USB HID-mouse application
00026  * on the AVR32 microcontroller.
00027  *
00028  * As the AVR32 implements a device/host USB controller, the embedded application can operate
00029  * in one of the following USB operating modes:
00030  *   - USB device;
00031  *   - USB reduced-host controller;
00032  *   - USB dual-role device (depending on the ID pin).
00033  *
00034  * To optimize embedded code/RAM size and reduce the number of source modules, the application can be
00035  * configured to use one and only one of these operating modes.
00036  *
00037  * \section sample About the Sample Application
00038  * By default the sample code is delivered with a simple preconfigured dual-role USB application.
00039  * It means that the code generated allows to operate as a device or a host depending on the USB ID pin:
00040  *
00041  *   - Attached to a mini-B plug (ID pin unconnected) the application will be used in the device operating mode.
00042  * Thus, the application can be connected to a system host (e.g. a PC) to operate as a USB mouse device:
00043  *
00044  * \image html appli_evk1100_device.jpg "EVK1100 USB Device Mode"
00045  * \image html appli_evk1101_device.jpg "EVK1101 USB Device Mode"
00046  *
00047  * The mouse can be controlled thanks to the joystick and the buttons (shown in a red box in the above pictures).
00048  *
00049  *   - Attached to a mini-A plug (ID pin tied to ground) the application will operate in reduced-host mode.
00050  * This mode allows to connect a USB mouse device:
00051  *
00052  * \image html appli_evk1100_host.jpg "EVK1100 USB Host Mode"
00053  * \image html appli_evk1101_host.jpg "EVK1101 USB Host Mode"
00054  *
00055  * In this mode, a mouse ascii pointer is output on the USART1 connector. Open a serial terminal (e.g. HyperTerminal under
00056  * Windows systems or minicom under Linux systems; USART settings: 57600 bauds, 8-bit data, no parity bit,
00057  * 1 stop bit, no flow control). Make sure that the  serial terminal takes in charge the ANSI codes.
00058  *
00059  * \image html terminal.JPG "Debug output in terminal"
00060  *
00061  * \section device_use Using the USB Device Mode
00062  * Connect the application to a USB host (e.g. a PC) with a mini-B (embedded side) to A (PC host side) cable.
00063  * The application will behave as a mouse peripheral. The mouse can be controlled thanks to the joystick and
00064  * the buttons of the EVK110x.
00065  *
00066  * \section host_use Using the USB Host Mode
00067  * Connect the application to a USB mouse device. The application will behave as a USB mouse
00068  * reduced host. Leds will blink according to the mouse eactivity and a mouse ascii pointer is output on the USART1 connector.
00069  *
00070  * @section arch Architecture
00071  * As illustrated in the figure bellow, the application entry point is located is the hid_example.c file.
00072  * The application can be ran using the FreeRTOS operating system, or in standalone mode.
00073  * In the standalone mode, the main function runs the usb_task and device_mouse_hid_task tasks in an infinite loop.
00074  *
00075  * The application is based on three different tasks:
00076  * - The usb_task  (usb_task.c associated source file), is the task performing the USB low level
00077  * enumeration process in device mode.
00078  * Once this task has detected that the usb connection is fully operationnal, it updates different status flags
00079  * that can be check within the high level application tasks.
00080  * - The device_mouse_hid_task task performs the high level device application operation.
00081  * Once the device is fully enumerated (DEVICE SET_CONFIGURATION request received), the task
00082  * transmit data (reports) on its IN endpoint.
00083  * - The host_mouse_hid_task task performs the high level host application operation.
00084  * Once a device is connected and fully enumerated, the task
00085  * receive data (reports) on its IN endpoint.
00086  *
00087  * \image html arch_full.jpg "Architecture Overview"
00088  *
00089  * \section files Main Files

00093  * - host_mouse_hid_task.h : high level host mouse application operation header
