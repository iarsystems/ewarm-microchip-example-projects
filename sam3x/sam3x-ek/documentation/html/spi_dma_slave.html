SPI Slave Example

Purpose

This example uses Serial Peripheral Interface (SPI1) in slave mode to communicate to another interface (SPI0) in master mode .

Requirements

This package can be used with at91sam3x-ek board by switch off the data flash. Please connect the SPI pins :

  • SPI0(MASTER)--SPI1(SLAVE)
  • SPI0_NPCS0(PA28)--SPI1_NPCS0(PE31)
  • SPI1_MISO(PA25)--SPI1_MISO(PAE28)
  • SPI1_MOSI(PA26)--SPI1_MOSI(PE29)
  • SPI0_SPCK(PA27)--SPI1_SPCK(PE30)

Descriptions

This example shows control of the SPI slave, how to configure and transfer data with SPI slave. By default, example runs in SPI slave mode, waiting SPI slave & UART inputs.

The code can be roughly broken down as follows:

  • 't' will start SPI transfer test
    1. Configure SPI as master, setup SPI clock.
    2. Send 4 byte CMD_TEST to indicate the start of test.
    3. Send several 64 bytes block, on transmitting next block, the content of last block is returned.
    4. Send CMD_STATUS command and waiting for the status reprots from slave.
    5. Send CMD_END command to indicate the end of test.
  • Setup SPI clock for master.

Usage

  1. Compile the application.
  2. Connect the DBGU port of the evaluation board to the computer and open it in a terminal.
    • Settings: 115200 bauds, 8 bits, 1 stop bit, no parity, no flow control.
  3. Download the program inside the evaluation board and run it. Please refer to SAM-BA User Guide, the GNU-Based Software Development application note or to the IAR EWARM User Guide, depending on your chosen solution.
  4. Upon startup, the application will output the following line on the DBGU: 
         -- Spi slave Example  xxx --
     -- AT91xxxxxx-xx
     -- Compiled: xxx xx xxxx xx:xx:xx --
  5. The following traces detail operations on the spl slave example, displaying success or error messages depending on the results of the commands.
