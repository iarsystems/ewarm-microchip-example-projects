Example description

Analog-to-Digital Converter Interface Example

DESCRIPTION
===========
  The adcife_example demonstrates how to set up the module and perform conversions.

  This application shows how to use ADCIFE using the several modes:
  with/without PDC, several types of trigger (Software, Continuous,
  Internal Timer, etc.),gain selection. The SAM4L controller initialize
  the ADCIFE and display the converted value in the terminal. Users can
  also select different modes by configuration menu in the terminal.

  The application will output converted value to the terminal program and display
  a menu for users to set different modes..

COMPATIBILITY
=============
  The example project is compatible with Microchip SAM4L-EK evaluation board.

CONFIGURATION
=============
  Debug   - The configuration runs the example from the internal flash memory.
  Release - The configuration runs the example from the internal flash memory 
            with no debug info.

  We select the internal DAC = 826mv and scaled VCC(VCC/10 = 330mv) as positive
  input to ADCIFE.

  Please set the following settings in your terminal software:

  Baud rate : 115200
  Data bits : 8
  Parity    : none
  Stop bits : 1

  Use the virtual JLink CDC UART Port number, when configuring the COM port.

GETTING STARTED
===============
  1) Choose a confguration.
  2) Build the example.
  3) Open the JLink CDC UART Port with a terminal program on your PC.
  4) Connect headphones to the board.
  5) Download and run the example.
