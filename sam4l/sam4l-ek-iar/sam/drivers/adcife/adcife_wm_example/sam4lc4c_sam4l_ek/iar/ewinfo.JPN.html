ADCIFE Window Monitor Example

DESCRIPTION
===========
  The adcife_window_monitor example demonstrates how to use window monitor mode.

  This example uses continuous mode for ADCIFE conversion. It would trigger the ADCIFE
  to start a conversion on the given channel which is DAC internal selection.

  With the user interface, DAC output voltage, window threshold and mode could be set.
  After setting, it will start a conversion and enable window monitor interrupt.
  If conversion result fall into comparison window, then it will generate relative
  interrupt and display information in terminal. 

COMPATIBILITY
=============
  The example project is compatible with Microchip SAM4L-EK evaluation board.

CONFIGURATION
=============
  Debug   - The configuration runs the example from the internal flash memory.
  Release - The configuration runs the example from the internal flash memory 
            with no debug info.

  We select the internal DAC = 826mv and scaled VCC(VCC/10 = 330mv) as positive
  input to ADCIFE.

  Please set the following settings in your terminal software:

  Baud rate : 115200
  Data bits : 8
  Parity    : none
  Stop bits : 1

  Use the virtual JLink CDC UART Port number, when configuring the COM port.

GETTING STARTED
===============
  1) Choose a confguration.
  2) Build the example.
  3) Open the JLink CDC UART Port with a terminal program on your PC.
  4) Download and run the example.
