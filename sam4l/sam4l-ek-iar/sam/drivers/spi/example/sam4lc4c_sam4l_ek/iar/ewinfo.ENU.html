Example description

Serial Peripheral Interface (SPI) example for SAM.

DESCRIPTION
===========
  This example uses Serial Peripheral Interface (SPI) of one EK board in
  slave mode to communicate with another EK board's SPI in master mode.

  This package can be used with two SAM evaluation kits boards.
  Please connect the SPI pins from one board to another.

  This example shows control of the SPI, and how to configure and
  transfer data with SPI. By default, example runs in SPI slave mode,
  waiting SPI slave & UART inputs.
 
  The code can be roughly broken down as follows:

  - 't' will start SPI transfer test.

  - Configure SPI as master, and set up SPI clock.
  - Send 4-byte CMD_TEST to indicate the start of test.
  - Send several 64-byte blocks, and after transmitting the next block, the
  - content of the last block is returned and checked.
  - Send CMD_STATUS command and wait for the status reports from slave.
  - Send CMD_END command to indicate the end of test.

  - Setup SPI clock for master.


COMPATIBILITY
=============
  The example project is compatible with Microchip SAM4L-EK evaluation board.

CONFIGURATION
=============
  Debug   - The configuration runs the example from the internal flash memory.
  Release - The configuration runs the example from the internal flash memory 
            with no debug info.

  Please set the following settings in your terminal software:

  Baud rate : 115200
  Data bits : 8
  Parity    : none
  Stop bits : 1

  Use the virtual JLink CDC UART Port number, when configuring the COM port.

GETTING STARTED
===============
  1) Choose a confguration.
  2) Build the example.
  3) Open the JLink CDC UART Port with a terminal program on your PC.
  4) Download and run the example.
